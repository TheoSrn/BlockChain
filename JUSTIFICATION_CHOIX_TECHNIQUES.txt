1. CHOIX DE LA BLOCKCHAIN : ETHEREUM
═══════════════════════════════════════════════════════════════════════════════

✅ J'ai choisi Ethereum (testnet Sepolia)

Pourquoi Ethereum ?

1. C'est la blockchain la plus connue pour les smart contracts
   • Beaucoup de documentation et tutoriels disponibles
   • Grande communauté pour demander de l'aide
   
2. Uniswap est déjà disponible sur Ethereum
   • Pas besoin de créer mon propre système d'échange
   • Facile à intégrer dans mon projet
   
3. OpenZeppelin offre des contrats sécurisés
   • Code déjà testé par des experts
   • Je peux réutiliser des contrats fiables
   
4. Outils pratiques
   • Hardhat pour développer facilement
   • MetaMask que tout le monde connaît
   • Compatible avec mes connaissances en JavaScript/TypeScript
   
5. Système KYC facile à implémenter
   • Je peux vérifier l'identité dans les smart contracts
   • Conforme aux règles légales

Pourquoi pas XRP Ledger ?
   - Moins de ressources pour apprendre
   - DEX moins développé
   - Moins d'outils disponibles
   - Plus compliqué pour débuter

Conclusion : Ethereum était le choix le plus simple pour commencer

2. MES SMART CONTRACTS EN SOLIDITY
═══════════════════════════════════════════════════════════════════════════════

✅ Langage utilisé : Solidity 0.8.20

J'ai créé ces contrats principaux :

2.1 Factory.sol - Pour créer les assets
   
   Ce que ça fait :
   • Crée automatiquement un ERC20 + un NFT + un Pool
   • Vérifie que l'utilisateur est whitelisté avant de créer
   • Économise du gas avec le système de Clones
   
   Pourquoi c'est utile :
   → Tout est créé en une seule fois
   → Pas besoin de déployer 3 contrats séparément
   → Plus facile à gérer

2.2 AssetERC20.sol - Tokens divisibles
   
   Basé sur ERC-20 d'OpenZeppelin
   
   Ce que ça permet :
   • Diviser un bien en plusieurs parts (comme des actions)
   • Transférer des parts entre utilisateurs
   • Vérifier le KYC à chaque transfert
   
   Exemple d'utilisation :
   - Un immeuble de 1,000,000 EUR
   - Je crée 1,000,000 tokens (1 token = 1 EUR)
   - Les gens peuvent acheter 100 tokens (100 EUR) au lieu de tout l'immeuble
   
   Pourquoi ERC-20 :
   → Standard connu et utilisé partout
   → Fonctionne avec Uniswap directement
   → Compatible avec MetaMask

2.3 AssetNFT.sol - Tokens uniques
   
   Basé sur ERC-721 d'OpenZeppelin
   
   Ce que ça représente :
   • Un bien unique (villa, diamant, œuvre d'art)
   • 1 seul NFT = 1 seul propriétaire
   • Métadonnées stockées (adresse, surface, prix, etc.)
   
   Exemple :
   - Villa à Monaco = NFT #001
   - Seul le propriétaire du NFT possède la villa
   - Impossible de diviser en plusieurs parts
   
   Pourquoi ERC-721 :
   → Standard universel pour les NFTs
   → Compatible avec OpenSea
   → Facile à comprendre

2.4 TradingPool.sol - Pour trader avec KYC
   
   Ce que ça fait :
   • Vérifie le KYC avant chaque trade
   • Utilise Uniswap pour les échanges
   • Protège contre les attaques
   
   Comment ça marche :
   1. User demande d'échanger des tokens
   2. Le contrat vérifie s'il est whitelisté
   3. Si oui, le contrat fait l'échange via Uniswap
   4. Si non, le trade est bloqué
   
   Pourquoi c'est important :
   → Impossible de trader sans être vérifié
   → Respecte les lois (KYC obligatoire)
   → Sécurisé

2.5 KYC.sol - Gestion des identités
   
   Fonctions simples :
   • Whitelist : liste des gens autorisés
   • Blacklist : liste des gens interdits
   • Vérification à chaque action
   
   Qui peut modifier :
   → Seulement l'administrateur
   → Pas les utilisateurs normaux
   
   Pourquoi :
   → Conformité légale
   → Éviter les fraudes

2.6 Oracle.sol - Prix des assets
   
   Ce qu'il stocke :
   • Prix de chaque asset
   • Date de la dernière mise à jour
   • Devise (USD, EUR, etc.)
   
   Utilisation :
   → L'admin met à jour les prix
   → Le frontend affiche les prix actuels
   → Calcul de la valeur du portfolio

Tokens retournés au user

5. TOKENS FONGIBLES (ERC-20) - ASSETS DIVISIBLES
═══════════════════════════════════════════════════════════════════════════════

✅ J'utilise le standard ERC-20 (OpenZeppelin)
✅ Mon contrat : AssetERC20.sol

Qu'est-ce qu'un token fongible ?

C'est comme de l'argent : tous les tokens sont identiques
• 1 token = 1 token (interchangeable)
• On peut diviser en petites parts
• Utilisé pour les biens qu'on veut fractionner

Exemples concrets :

Exemple 1 : Immeuble à Paris
• Valeur totale : 5,000,000 EUR
• Je crée 5,000,000 tokens (1 token = 1 EUR)
• Un investisseur peut acheter juste 10,000 tokens (10,000 EUR)
• Pas besoin d'acheter tout l'immeuble
→ Permet aux petits investisseurs de participer

Exemple 2 : Actions d'entreprise
• Startup valorisée 10,000,000 USD
• 100,000 tokens créés (1 token = 100 USD)
• Founders gardent 50,000 tokens
• 30,000 tokens pour investisseurs
• 20,000 tokens pour employés
→ Facile à distribuer et échanger

Exemple 3 : Or physique
• 100 kg d'or dans un coffre
• 6,000,000 tokens (1 token = 1 EUR d'or)
• Les gens achètent des parts sans avoir l'or chez eux
→ Trading facile 24/7

Pourquoi ERC-20 ?

✓ Standard connu de tous
✓ Fonctionne avec MetaMask, Uniswap, etc.
✓ Peut être divisé jusqu'à 18 décimales
✓ Facile à échanger

Mon système KYC :

À chaque transfert, le contrat vérifie :
• L'envoyeur est-il whitelisté ?
• Le receveur est-il whitelisté ?
• Quelqu'un est-il blacklisté ?

Si non → Transfer bloqué
Si oui → Transfer ok

Pourquoi pas XRPL MPT ?
- Moins connu
- Moins d'outils disponibles
- Plus compliqué pour débuter

6. TOKENS NON-FONGIBLES (ERC-721) - ASSETS UNIQUES
═══════════════════════════════════════════════════════════════════════════════

✅ J'utilise le standard ERC-721 (OpenZeppelin)
✅ Mon contrat : AssetNFT.sol

Qu'est-ce qu'un token non-fongible (NFT) ?

C'est un token unique, impossible à diviser
• 1 NFT = 1 bien unique
• Chaque NFT est différent
• Utilisé pour biens qu'on ne peut pas fractionner

Informations stockées dans le NFT :

• Location : où se trouve le bien
• Surface : taille en m²
• Valeur estimée : prix
• Description : détails
• Documents : lien vers preuves (IPFS)

Exemples concrets :

Exemple 1 : Villa à Monaco
• NFT #001 = Villa complète
• Location : "Monaco, Avenue Princesse Grace"
• Surface : 500 m²
• Valeur : 10,000,000 EUR
• Owner : 1 seule personne possède le NFT
→ Titre de propriété numérique

Exemple 2 : Œuvre d'art
• NFT #042 = Tableau "Sunset Digital"
• Valeur : 50,000 USD
• Artist peut toucher 10% sur les reventes
• Compatible avec OpenSea
→ Certificat d'authenticité on-chain

Exemple 3 : Diamant
• NFT #123 = Diamant 5 carats
• Certificat GIA lié au NFT
• Stocké dans coffre sécurisé à Genève
• Owner peut récupérer le diamant physique
→ Trading du diamant sans le déplacer

Pourquoi ERC-721 ?

✓ Standard universel pour les NFTs
✓ Compatible avec OpenSea, Rarible, etc.
✓ Fonctionne avec tous les wallets NFT
✓ Historique de propriété transparent

Mon système KYC :

Comme pour les ERC-20, chaque transfert vérifie :
• Les 2 parties sont whitelistées ?
• Personne n'est blacklisté ?

Si non → Transfer bloqué

Pourquoi pas XRPL XLS-20 ?
- Moins de marketplaces disponibles
- Moins d'outils
- Communauté plus petite
